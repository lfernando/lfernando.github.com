Frameworks CSS

Feb 13th, 2012

Quando ouvimos a palavra framework pensamos sempre em frameworks de programação, como Rails, DJango, CakePHP dentre outros. Porém já faz um tempo que comecei ouvi falar em Frameworks de CSS. Assim como os frameworks de programação os de CSS buscam agilizar e padronizar nosso trabalho, esses frameworks possuem recursos muito uteis como o “reset” (que reseta o estilo padrão das principais tags, que são interpretadas de maneira diferente por cada browser), o “Grid” (que ajuda posicionar os elementos da aplicação).

Como trabalho com Web e normalmente faço tanto o backend quanto o frontend decidi experimentar esses frameworks e ver se eles realmente oferecem alguma vantagem no desenvolvimento de um site ou aplicação web.

Dentre os frameworks existentes estou listando abaixo os mais conhecidos e os quais achei mais interessantes:

  • Foundation Zurb
  • Bootstrap Twitter
  • Fbootstrapp
  • Blueprint
  • 960 Grid
  • Formee

Os quatro primeiros são framework completos e possuem recursos como o reset, grid e formatação para o formulários e botões, etc. Enquanto os dois últimos são mais especificos o 960 Grid é específico pra Grid e o Formee específico para formatação de formulários.

Após ler sobre cada um e avaliando minhas necessidades, os que mais me agradaram foi o Zurb e o Bootstrap Twitter, no entanto decidi usar o Zurb, pois o bootstrap do twitter deixa a aplicação com aparencia do Twitter o que tiraria um pouco da personalidade da minha aplicação, enquanto o Zurb é mais simples e me permite personalizá-lo facilmente. Escolhido o Framework comecei a utilizá-lo e logo no começo já percebi as vantagens de utilizar este tipo de framework, ele esta agilizando muito meu trabalho. Assim que terminar minha aplicação posto aqui o link para verem o resultado final.
